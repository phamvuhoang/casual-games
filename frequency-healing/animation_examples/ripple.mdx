## Usage

```tsx
import { Ripple } from "@/components/ui/ripple";

export default function CallToAction() {
  return (
    <div className="relative">
      {/* Your button or content */}
      <button className="relative z-10 px-8 py-4 bg-blue-600 text-white rounded-lg">
        Get Started
      </button>
      
      {/* Ripple attention effect */}
      <Ripple 
        mainCircleSize={200}
        mainCircleOpacity={0.2}
        numCircles={6}
      />
    </div>
  );
}
```

## Required CSS Animation

**Important**: Add this CSS animation to your global styles for ripples to work:

```css
@keyframes ripple {
  0%, 100% {
    transform: translate(-50%, -50%) scale(1);
  }
  50% {
    transform: translate(-50%, -50%) scale(0.9);
  }
}

.animate-ripple {
  animation: ripple var(--duration, 2s) ease calc(var(--i, 0) * 0.2s) infinite;
}
```

## API Reference

### Ripple

Main component for concentric circle attention and focus effects.

| Prop                | Type     | Default | Description                                               |
| ------------------- | -------- | ------- | --------------------------------------------------------- |
| `mainCircleSize`    | `number` | `210`   | Diameter of the main circle in pixels (100-400 recommended) |
| `mainCircleOpacity` | `number` | `0.24`  | Opacity of innermost circle (others fade progressively)  |
| `numCircles`        | `number` | `8`     | Number of concentric circles (3-12 optimal range)        |
| `className`         | `string` | -       | Additional CSS classes for container styling              |

### Animation Timing System

Each circle animates with carefully calculated delays:

```tsx
// Circle delays create wave propagation
Circle 1: 0s delay      (center, most visible)
Circle 2: 0.2s delay
Circle 3: 0.4s delay
Circle 4: 0.6s delay    (outer circles fade to nearly invisible)
...
```

### Ripple Intensity Guidelines

```tsx
// Subtle background emphasis
mainCircleSize={150} mainCircleOpacity={0.1} numCircles={4}
```

## Common gotchas

**Missing CSS animation**: The component won't animate without the required `@keyframes ripple` in your global styles. Circles will appear static without this crucial animation.

**Overflow container required**: Ripples extend beyond their container bounds. Always use `overflow-hidden` on the parent element to prevent layout issues and unwanted scrollbars.

**Z-index layering conflicts**: Ripples appear behind content by default. If your content disappears, ensure it has `relative z-10` or higher positioning.

**Performance with many circles**: Each circle is an animated element. Very high `numCircles` values (15+) might impact performance on older devices. 8 circles is the optimal balance.

**Animation timing accumulation**: Multiple ripples on the same page all animate simultaneously. Consider staggering component mounting or using different durations to avoid synchronized pulsing.

**Accessibility considerations**: While subtle, ripples are still motion effects. Consider respecting `prefers-reduced-motion` for users with motion sensitivity.

## Questions developers actually ask

<Accordions type="single">
  <Accordion id="css-setup" title="Why aren't my ripples animating?">
    The component requires CSS keyframes in your global styles. Add the `@keyframes ripple` animation and `.animate-ripple` class to your CSS file. Without this, circles appear static.
  </Accordion>

  <Accordion id="timing-control" title="How do I change the animation speed?">
    Use the CSS custom property `--duration` to control timing: `.animate-ripple { --duration: 3s; }` for slower, or `--duration: 1s` for faster pulsing.
  </Accordion>

  <Accordion id="color-customization" title="Can I change ripple colors or make them more visible?">
    Ripples inherit border colors from your theme. Override with CSS: `.ripple-circle { border-color: #your-color; }` or increase `mainCircleOpacity` for more visibility.
  </Accordion>

  <Accordion id="optimal-settings" title="What are the best settings for different use cases?">
    Buttons: size=180, opacity=0.2, circles=5. Cards: size=120, opacity=0.15, circles=4. Hero sections: size=300, opacity=0.3, circles=8. Adjust based on your design needs.
  </Accordion>

  <Accordion id="multiple-ripples" title="Can I use multiple ripples on the same page?">
    Yes, but they'll all pulse in sync. For varied timing, consider adding CSS delays or using different `--duration` values for each ripple instance.
  </Accordion>

  <Accordion id="performance-optimization" title="How do I optimize performance with many ripples?">
    Reduce `numCircles` to 4-6, use lower `mainCircleOpacity` values (0.1-0.2), and consider disabling ripples on mobile devices if battery life is a concern.
  </Accordion>
</Accordions>
